# Current Focus — Plugin PowerBuilder 2025 para VS Code

## 1. Propósito

Este documento refleja el foco operativo actual del proyecto.

Debe responder siempre a estas preguntas:

- qué estamos cerrando ahora,
- por qué es prioritario,
- qué no debemos tocar salvo necesidad,
- y cuál es el siguiente paso natural.

---

## 2. Foco actual

### Prioridad operativa

Fortalecer la base del plugin para que la carga, la activación y el esqueleto LSP sean profesionales, rápidos y fiables.

### Objetivos inmediatos

1. consolidar activación perezosa,
2. mantener cliente fino y servidor claro,
3. medir y proteger tiempos de carga,
4. priorizar el archivo abierto,
5. preparar índice incremental sin sobrecargar la base,
6. mantener toda documentación canónica alineada.

---

## 3. Qué sí debe hacerse ahora

- endurecer bootstrap del cliente,
- estabilizar ciclo de vida del servidor,
- eliminar trabajo pesado del arranque,
- reforzar estructura modular,
- preparar métricas y validaciones básicas,
- y dejar docs actualizadas con el estado real.

---

## 4. Qué no debe hacerse ahora salvo causa clara

- reabrir arquitectura general sin motivo,
- meter features vistosas antes de consolidar base,
- ampliar demasiado la superficie funcional,
- introducir complejidad innecesaria en el cliente,
- cerrar trabajo sin validación y sin documentación.

---

## 5. Riesgos actuales a vigilar

- exceso de lógica en el Extension Host,
- activación demasiado amplia,
- indexación temprana agresiva,
- crecimiento desordenado de módulos,
- y documentación desalineada con implementación real.

---

## 6. Siguiente paso natural

El siguiente paso natural del proyecto es:

1. cerrar la base de activación y carga,
2. dejar el wiring LSP limpio,
3. introducir el esqueleto de indexación incremental,
4. y después avanzar al núcleo semántico útil del archivo activo.

---

## 7. Condición para mover el foco

El foco actual solo debe cambiar cuando:

- la base esté estable,
- existan mediciones mínimas de rendimiento,
- el arranque no bloquee el editor,
- y la estructura documental quede alineada con el estado del repositorio.
